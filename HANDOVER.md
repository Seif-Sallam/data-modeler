# Data Modeler вАФ Handover

A local, single-user web app for drafting database schemas visually and exporting them to SQLAlchemy Python tables (and Markdown reports). Single-process Flask backend + vanilla-JS SPA. Designed to run offline (one CDN dep вАФ see below).

---

## Run it

```bash
./run.sh
```

That's literally `uv run --with flask python app.py`. Server listens on `http://127.0.0.1:5050`. Drafts persist as JSON in `drafts/`; every overwrite first copies the old file to `backups/<draft>.<timestamp>.json`.

`pyproject.toml` declares one dependency: `flask>=3.0.0`. Python 3.10+.

**One external runtime dep** вАФ `html2canvas@1.4.1` is loaded from jsdelivr in the page `<head>`, used only by the рЯУЄ Snapshot button. The rest of the app works offline. If you need a fully air-gapped build, download the script into `static/` and update the `<script src>` tag.

## Data model (the JSON shape)

A draft is one object. All fields not listed here are unowned by code and round-trip freely.

```jsonc
{
  "name": "untitled",
  "tabs": [
    {
      "id": "abc1234",
      "name": "Tab 1",
      "description": "optional free text",
      "view": { "panX": 0, "panY": 0, "zoom": 1 },   // per-tab persisted view
      "tables": [
        {
          "id": "xyz9876",
          "table_name": "user",          // snake_case; class name is CamelCase of this
          "x": 200, "y": 200,            // canvas position
          "w": 380,                      // optional table card width (corner-resize handle)
          "colW": { "name": 140, "type": 120 },  // per-column header widths
          "collapsed": false,
          "comment": "free markdown text",
          "comment_pinned": false,
          "columns": [
            {
              "id": "c123",
              "name": "id_user",
              "type": "BIGINT",          // canonical types: BIGINT, INT, SMALLINT, TINYINT, String(N), TEXT, DATETIME, DATE, BOOLEAN, JSON, DECIMAL(p,s), CCY
              "primary_key": true,
              "nullable": false,
              "unique": false,
              "index": false,
              "server_default": "",
              "comment": "free markdown text",
              "comment_pinned": false,
              "no_autoincrement": false
            }
          ],
          "constraints": [
            { "kind": "unique" | "index", "name": "uniq_x", "columns": ["a","b"] }
          ]
        }
      ],
      "notes": [
        {
          "id": "n1", "x": 50, "y": 50,
          "title": "",                   // optional; placeholder "рЯУЭ Note" shown when empty
          "text": "**markdown** body",   // rendered live in the preview pane
          "collapsed": false,
          "w": 240, "h": 160,            // expanded size
          "wMin": 180, "hMin": 80        // minimized size (independent from expanded)
        }
      ],
      "links": [
        { "fromTable": "id", "fromCol": "id", "toTable": "id", "toCol": "id", "hidden": false }
      ]
    }
  ]
}
```

**Authoritative rules**

- Every entity has an `id` (generated by `uid()` in JS). `normalize(state)` on load back-fills missing ids вАФ never remove that function.
- Class name is derived (`toCamel(table_name)`) and is **not** stored.
- Links are "imaginary" вАФ pure UI affordances. They never become foreign keys in the export.
- Type strings in JSON are **canonical** (`String(255)`, `BIGINT`, вА¶). The codegen and parser convert to/from `sa.String(255)` etc. via `render_col_type` (app.py) and `_denorm_type` (app.py). Add new types in both places.

## Backend вАФ `app.py`

Thin Flask app, no blueprints, no DI.

| Route | Purpose |
|---|---|
| `GET /` | Serves `static/index.html`. |
| `GET /api/drafts` | List drafts (filename + modified mtime). |
| `GET /api/drafts/<name>` | Read one draft. |
| `PUT /api/drafts/<name>` | Save (writes a timestamped backup first). |
| `DELETE /api/drafts/<name>` | Delete. |
| `POST /api/duplicate` | Copy file. |
| `POST /api/rename` | Move file + update `data["name"]`. |
| `GET /api/export/<name>?tab=<id>&tables=<id,id>` | Python codegen (all / one tab / specific tables). |
| `GET /api/markdown/<name>?tab=<id>` | Markdown report вАФ includes a `**Notes**` section per tab. Blockquotes emit one `> ` per source line, no trailing `\`. |
| `POST /api/parse` `{source: str}` | Parse pasted Python class definitions back into tables. |

**Codegen вАФ `render_table`**
Emits `class X(Model):` with `__tablename__`, columns, then `__table_args__` for constraints. The header preamble (imports etc.) is intentionally **not** emitted.

**Parser вАФ `parse_python_tables`**
Regex-based. Handles `Column(...)`, `UniqueConstraint`, `Index`, `primary_key=`, `nullable=`, `unique=`, `index=`, `server_default=`. Extend both render and parser when you add a column property вАФ otherwise it becomes a one-way export.

**Markdown вАФ `render_markdown`**
One section per tab: tables вЖТ columns table вЖТ constraints вЖТ links вЖТ notes. Blockquotes (table comments, note bodies) emit each source line as its own `> ` line вАФ no trailing `\`. Pipes in cell content are escaped (`replace("|","\\|")`).

## Frontend вАФ `static/index.html`

One file, three concerns: CSS at top, HTML in the middle, all JS at the bottom in one `<script>`. State held in `state`; everything re-renders on change.

### Major sections (skim in this order)

1. **`:root, body.dark` CSS variables** вАФ every color comes from here. Don't hard-code hex.
2. **State + utilities** вАФ `state`, `currentTabId`, `zoom`, `panX/Y`, `selection`, `uid()`, `toCamel()`, `mdToHtml()`, `escHtml()`, `saveView()`, `restoreView()`.
3. **`render()` / `renderCanvas()`** вАФ `render()` checks if the tab changed and restores its view first. `renderCanvas()` wipes `#canvas`, re-renders tables + notes, then `renderLinks()`, `updateCoords()`, `applySelectionDom()`.
4. **`renderTable(t)`** вАФ collapsed (mini summary) vs expanded (`<table class="cols">`). Header has class name, вЦђ/вЦ£ minimize, вЭП duplicate, рЯЧС delete (click-to-arm 3s, browser `confirm()` is unreliable here). Two new resize affordances: a corner handle (`.resize-handle`) for `t.w`, and `.col-resize` strips on the Name/Type `<th>` for `t.colW.{name,type}`.
5. **`renderNote(n)`** вАФ header with auto-sized editable title input, `<textarea>` for editing, `.note-preview` div fed with `mdToHtml(text)`. Resize handle drags `n.w/n.h` when expanded, `n.wMin/n.hMin` when minimized. When minimized the title input is `readOnly` + `pointer-events:none` so it doesn't catch the caret.
6. **`renderLinks()`** вАФ SVG overlay. Anchors come from live `getBoundingClientRect()` вЖТ `toCanvas()` so links track zoom/pan/column-position/resize correctly. Faint X/Y axes at origin drawn here too.
7. **Selection + group drag** вАФ `selection` Set of `'table:<id>'`/`'note:<id>'` keys. **Ctrl/вМШ+click** toggles, **Ctrl/вМШ+A** selects all (sticky). Deselect happens on `mouseup` on empty canvas (deferred from `mousedown` so dragging a selected entity isn't misread). `startGroupDrag` moves the whole selection together via one global `mousemove` listener. New entity types must wire into `applySelectionDom`, `selectAll`, and `getSelectedEntities`.
8. **History** вАФ `commitHistory()` debounced 400 ms. `Cmd/Ctrl+Z` undo, `Shift+Z` or `Cmd+Y` redo.
9. **Modal** вАФ single `#modal` shared by exports, imports, link list. Always opened through `openModal({primaryLabel, primaryTitle, showCopy})`. Inline `#copyToast` next to the buttons is the *only* "Copied" indicator. `copyToClipboard()` falls back from `navigator.clipboard` to a hidden-textarea + `execCommand('copy')` so it works on Windows where the secure-context API may be missing.
10. **Markdown renderer вАФ `mdToHtml(text)`** вАФ supports `# ## ###`, `**bold**`, `*italic*`, `` `code` ``, fenced ``` ``` ``` blocks, `- / *` and `1.` lists, `> blockquotes`, `[link](url)`, `---` rules, and GFM pipe tables (`| h | h |` + `| --- | :-: |` separator with `:` alignment). Apply via `el.innerHTML = mdToHtml(text)` plus the shared `.md` CSS class. Used by note previews, table comments, and column comments.
11. **Comment buttons** вАФ table-comment buttons open the popover directly. Column-comment buttons (рЯТђ in the row actions) behave like this: **no comment yet** вЖТ single-click opens the editor. **Comment exists** вЖТ **single-click toggles pin** (debounced 260 ms so it doesn't fire on dblclick), **double-click opens the editor**. The pending pin-toggle is cancelled by a second click.
12. **Snapshot (рЯУЄ in `#viewControls`)** вАФ uses `html2canvas`. Builds an off-screen stage `<div>` containing cloned tables/notes/links at zoom=1, runs `html2canvas` at `scale:3`, downloads as `<draft>-<tab>.png`. The off-screen DOM approach sidesteps the macOS `<foreignObject>` taint problem that breaks pure-SVG rasterization.
13. **View persistence** вАФ every pan release, wheel zoom, keyboard zoom, and Reset calls `saveView()` which writes `panX/panY/zoom` onto `currentTab().view`. `saveView()` **does not** mark dirty вАФ view changes ride along on the next real save but never trigger the unsaved-changes prompt on their own. When you switch tabs (or load a draft), `render()` notices `currentTabId` changed and calls `restoreView()` before redrawing.
14. **Cursor-anchored zoom** вАФ the wheel handler on `#canvasWrap` computes the world point under the cursor and adjusts `panX/panY` so that point stays under the cursor after scaling. Keyboard zoom (`Cmd+=` / `Cmd+-`) is **not** cursor-anchored (no cursor position in a key event) and scales around the top-left.

### Conventions inside the SPA

- Mutate `state` directly, then `markDirty(); renderCanvas();`. Don't diff.
- `markDirty()` flips a flag and schedules an autosave + history commit.
- New optional fields persist for free (no schema validation on the backend). Add a default to `normalize()` only if older drafts need one.
- `requestAnimationFrame(() => вА¶focus())` is the working pattern for focusing freshly-rendered inputs.
- For drag handlers, prefer `startGroupDrag` over rolling your own вАФ multi-select-aware and zoom-aware.
- For local one-off drags (column resize, table corner resize), use the pattern: `mousedown` registers `mousemove` + `mouseup` listeners on `window`, then `mouseup` removes them. Do not attach permanent listeners that grow per render.

## Things to know before you change anything

- **No backend schema** вАФ drafts are free-form JSON. Adding fields is free; renaming fields breaks all existing user drafts. Don't rename.
- **Old drafts may be missing ids on tables/columns**. `normalize(state)` backfills them on load. Trust it; don't add separate compatibility shims.
- **Browser `confirm()` is unreliable** вАФ the user's "Do not show again" suppresses it. Use the click-to-arm pattern (button turns red, second click confirms). See рЯЧС delete in `renderTable`.
- **Link anchors must use `getBoundingClientRect + toCanvas()`** вАФ do not compute from `t.x` plus hard-coded offsets. Tables can be resized, columns reordered, column widths variable.
- **No bundler** вАФ every import is inline. The one exception is `html2canvas` via CDN. Don't introduce npm.
- **macOS taints `<foreignObject>` canvases.** The Snapshot button uses `html2canvas` on real on-page HTML for this reason. Don't try to "improve" it by going back to an SVG-with-foreignObject pipeline вАФ it will silently fail on Mac.
- **Don't add docstrings or comments unless the *why* is non-obvious.** Identifiers are well-named.

## How to add a new feature (recipes)

**A new column property (e.g. `auto_now`)**
1. Default it in `newCol()`.
2. Render an editor in the expanded row inside `renderTable`.
3. Emit it from `render_table` in `app.py`.
4. Recognize it in `parse_python_tables`.
5. (Optional) Show a badge in the collapsed `mini` view.

**A new export format**
1. Add `GET /api/<format>/<name>` in `app.py` returning `{<format>: "..."}`.
2. Add a menu item under the Export dropdown in `index.html`.
3. Reuse the modal: `openModal({showCopy:true})` + put text in `<pre id="exportPre">` + set `window._lastExport`.

**A new entity type on canvas (e.g. a "group" box)**
1. Add `tab.groups = []` and `normalize` it.
2. Write `renderGroup(g)` mirroring `renderNote`'s drag/resize/select pattern.
3. Append it to `applySelectionDom`, `selectAll`, `getSelectedEntities`, and the `canvasWrap` mousedown's entity check.
4. If links should connect to it, extend `renderLinks` and the anchor logic.
5. If the snapshot should include it, add the selector to the `elems` query in `snapshotCanvas`.

**A new surface that renders markdown**
- `innerHTML = mdToHtml(text)` on the element, then add the `md` class вАФ the markdown CSS rules key off `.md, .note-preview` selectors.

## Known caveats / loose ends

- **`html2canvas` from CDN** вАФ the Snapshot button needs it. If the page is loaded fully offline before the script ever cached, the button fails with a status message. Self-host the file if you need true air-gap.
- **Python parser is regex-based.** Anything past what `render_table` produces (computed defaults, hybrid properties) won't round-trip.
- **No auth, no multi-user merge, no diff view.** Intentional.
- Soft optional fields older drafts won't have: `notes`, `links[].hidden`, `colW`, `w/h/wMin/hMin`, `comment_pinned`, `view`, `title` on notes, `comment` on columns. UI tolerates absence.

## When something looks wrong

- **Old draft missing buttons / can't delete a table** вЖТ check that `normalize(state)` ran on load.
- **Link line floating in wrong place** вЖТ `renderLinks` was probably called without applying zoom/pan to `#linkSvg`'s transform. Look at the SVG `style.transform` line near the top of `renderLinks`.
- **Export produces garbage type** вЖТ add the type to both `render_col_type` (Python emit) and `_denorm_type` (JSON canonicalize).
- **Frontend stops rendering after a UI change** вЖТ almost always a thrown error in an event handler. Open the browser console.
- **Snapshot fails on Mac** вЖТ confirm `html2canvas` is loaded (DevTools вЖТ Network). If yes and it still fails, the off-screen stage may be too big вАФ cap the bbox dimensions or reduce `scale` from 3 to 2.
- **Tab view reset on switch** вЖТ `_lastRenderedTabId` is probably out of sync; check that `loadDraft` resets it to `null` so `render()` calls `restoreView()`.

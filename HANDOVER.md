# Data Modeler — Handover

A local, single-user web app for drafting database schemas visually and exporting them to SQLAlchemy Python tables (and Markdown reports). Single-process Flask backend + vanilla-JS SPA. Designed to run offline.

---

## Run it

```bash
./run.sh
```

That's literally `uv run --with flask python app.py`. Server listens on `http://127.0.0.1:5050`. Drafts persist as JSON in `drafts/`; every overwrite first copies the old file to `backups/<draft>.<timestamp>.json`.

`pyproject.toml` declares one dependency: `flask>=3.0.0`. Python 3.10+.

```
pyproject.toml      Project metadata + flask dep.
```

There is no JS framework, no bundler, no test suite, no `__init__.py`. Keep it that way unless you have a strong reason.

## Data model (the JSON shape)

A draft is one object with this shape (everything not listed is optional and unowned by code — it round-trips):

```jsonc
{
  "name": "untitled",
  "tabs": [
    {
      "id": "abc1234",
      "name": "Tab 1",
      "description": "optional free text",
      "tables": [
        {
          "id": "xyz9876",
          "table_name": "user",          // snake_case; class name is CamelCase of this
          "x": 200, "y": 200,            // canvas position
          "w": 380,                      // optional table card width
          "colW": { "name": 140, "type": 120 },  // optional per-column widths
          "collapsed": false,
          "comment": "free text",
          "comment_pinned": false,
          "columns": [
            {
              "id": "c123",
              "name": "id_user",
              "type": "BIGINT",          // canonical types: BIGINT, INT, SMALLINT, TINYINT, String(N), TEXT, DATETIME, DATE, BOOLEAN, JSON, DECIMAL(p,s), CCY
              "primary_key": true,
              "nullable": false,
              "unique": false,
              "index": false,
              "server_default": "",
              "comment": "",
              "no_autoincrement": false
            }
          ],
          "constraints": [
            { "kind": "unique" | "index", "name": "uniq_x", "columns": ["a","b"] }
          ]
        }
      ],
      "notes": [
        {
          "id": "n1", "x": 50, "y": 50,
          "title": "",                   // optional, placeholder shows "📝 Note"
          "text": "markdown body",
          "collapsed": false,
          "w": 240, "h": 160,            // expanded size
          "wMin": 180, "hMin": 80        // minimized size
        }
      ],
      "links": [
        { "fromTable": "id", "fromCol": "id", "toTable": "id", "toCol": "id", "hidden": false }
      ]
    }
  ]
}
```

**Authoritative rules**

- Every entity has an `id` (generated by `uid()` in JS). `normalize(state)` on load back-fills missing ids — never remove that function.
- Class name is derived (`toCamel(table_name)`) and is **not** stored.
- Links are "imaginary" — pure UI affordances. They never become foreign keys in the export.
- Type strings in JSON are **canonical** (`String(255)`, `BIGINT`, …). The codegen and parser convert to/from `sa.String(255)` etc. via `render_col_type` (app.py:135) and `_denorm_type` (app.py:256). Add new types in both places.

## Backend — `app.py`

Thin Flask app, no blueprints, no DI.

| Route | Purpose |
|---|---|
| `GET /` | Serves `static/index.html`. |
| `GET /api/drafts` | List drafts (filename + modified mtime). |
| `GET /api/drafts/<name>` | Read one draft. |
| `PUT /api/drafts/<name>` | Save (writes a timestamped backup first). |
| `DELETE /api/drafts/<name>` | Delete. |
| `POST /api/duplicate` | Copy file. |
| `POST /api/rename` | Move file + update `data["name"]`. |
| `GET /api/export/<name>?tab=<id>&tables=<id,id>` | Python codegen (all / one tab / specific tables). |
| `GET /api/markdown/<name>?tab=<id>` | Markdown report. |
| `POST /api/parse` `{source: str}` | Parse pasted Python class definitions back into tables. |

**Codegen — `render_table` (app.py:152)**
Emits `class X(Model):` with `__tablename__`, column definitions, then `__table_args__` for constraints. The header preamble (imports etc.) is intentionally **not** emitted — the user paste-merges into existing files.

**Parser — `parse_python_tables` (app.py:277)**
Regex-based, not an AST visitor. Handles `Column(...)`, `UniqueConstraint`, `Index`, `primary_key=`, `nullable=`, `unique=`, `index=`, `server_default=`. If you extend the column model, extend both `render_table` *and* the parser — failing to do so is a one-way export and bites users.

**Markdown — `render_markdown` (app.py:367)**
Emits one section per tab: tables with columns/constraints/links, then a `**Notes**` section. Multi-line blockquotes use trailing `\` line-breaks so newlines actually render. Pipe characters in cell content are escaped (`replace("|","\\|")`).

## Frontend — `static/index.html`

One file, three concerns: CSS at top, HTML in the middle (single body root), all JS at the bottom inside one `<script>`. State is held in a module-level `state` object; everything is re-rendered on change.

### Major sections (skim in this order)

1. **`:root, body.dark` CSS variables** (line ~9) — every color comes from here. Don't hard-code hex inside components.
2. **State + utilities** — `state`, `currentTabId`, `zoom`, `panX/Y`, `selection`, `uid()`, `toCamel()`, `mdToHtml()`, `escHtml()`.
3. **`renderCanvas()` (line ~520)** — wipes `#canvas`, re-renders all tables + notes, then `renderLinks()` and `applySelectionDom()`. Most state changes end with `markDirty(); renderCanvas();`.
4. **`renderTable(t)` (line ~671)** — builds one card. Two render paths: collapsed (mini summary rows) vs expanded (`<table class="cols">`). Header has class name (read-only), comment, ▬/▣ minimize, ❏ duplicate, 🗑 delete (click-to-arm 3s).
5. **`renderNote(n)`** — header with auto-sized editable title, `<textarea>` for editing, `.note-preview` div that gets `mdToHtml(text)`.
6. **`renderLinks()` (line ~590)** — SVG overlay. Anchors are computed from live `getBoundingClientRect()` + `toCanvas()` conversion, so links track zoom/pan/column-position correctly. The faint X/Y axes at origin are drawn here.
7. **Group drag + selection** — `selection` Set holds `'table:<id>'` / `'note:<id>'` keys. `startGroupDrag` moves the entire selection together via a global `mousemove` listener. Ctrl+click toggles; Ctrl+A selects all (sticky); mousedown on empty canvas + mouseup with no drag → clears.
8. **History** — `commitHistory()` is debounced 400 ms. `Cmd/Ctrl+Z` undo, `Shift+Z` or `Cmd+Y` redo.
9. **Modal** — single `#modal` shared by exports, imports, and link list. Always opened through `openModal({primaryLabel, showCopy})`. Inline `#copyToast` is the only "copied" indicator.

### Conventions inside the SPA

- Mutate `state` directly, then call `markDirty(); renderCanvas();`. Don't try to diff.
- `markDirty()` flips a dirty flag and schedules an autosave + history commit.
- Anything user-visible must round-trip through JSON save → backend → reload. If you add a field, just write it onto the state object — there is no schema validation on the backend, so it persists for free. **But** add it to `normalize()` if older drafts need a default value.
- `requestAnimationFrame(() => …focus())` is the working pattern for focusing freshly-rendered inputs (Shift+Enter add-column flow).
- For drag handlers, prefer `startGroupDrag` over rolling your own — it handles multi-select for free and is zoom-aware.

## Things to know before you change anything

- **No backend schema** — drafts are free-form JSON. Adding fields is free; renaming fields breaks all existing user drafts. Don't rename.
- **Old drafts may be missing ids on tables/columns**. `normalize(state)` backfills them on load. Trust it; don't add separate compatibility shims.
- **Browser `confirm()` is unreliable** — the user's "Do not show again" suppresses it. Use the click-to-arm pattern (button turns red for N seconds, second click confirms). See the 🗑 delete handler in `renderTable`.
- **Link anchors must use `getBoundingClientRect + toCanvas()`** — do not compute from `t.x` plus hard-coded offsets. The table card can be resized, columns can be reordered, columns have variable widths.
- **No bundler** — every import is inline. If you need a dep, paste it in or load it via CDN. Don't introduce npm.
- **Don't add docstrings or comments unless the *why* is non-obvious** (per the user's house rules). Identifiers are well-named.

## How to add a new feature (recipes)

**A new column property (e.g. `auto_now`)**
1. Default it in `newCol()`.
2. Render an editor for it in the expanded row inside `renderTable` (next to the existing checkboxes).
3. Emit it from `render_table` in `app.py`.
4. Recognize it in `parse_python_tables`.
5. (Optional) Show a badge in the collapsed `mini` view.

**A new export format**
1. Add a `GET /api/<format>/<name>` route in `app.py` that returns `{<format>: "..."}`.
2. Add a menu item under the Export dropdown in `index.html`.
3. Reuse the modal: `openModal({showCopy:true})` + put the text inside `<pre id="exportPre">` + set `window._lastExport`.

**A new entity type on canvas (e.g. a "group" box)**
1. Add `tab.groups = []` and `normalize` it.
2. Write `renderGroup(g)` mirroring `renderNote`'s drag/resize/select pattern.
3. Append it to `applySelectionDom`, `selectAll`, `getSelectedEntities`, and the `canvasWrap` mousedown's entity check.
4. If links should connect to it, extend `renderLinks` and the anchor logic.

## Known caveats / loose ends

- `window.addEventListener('mousemove'/'mouseup', …)` is called inside per-entity render functions and per-drag setups. The shared group-drag listeners attached at module top are the canonical path now; the per-entity inline drag has been migrated, but column-resize and corner-resize add their own `addEventListener` pairs which they tear down on `mouseup`. New drag features should follow that local-listener pattern (add on mousedown, remove on mouseup) — do not attach permanent module-level listeners that grow per render.
- The Python parser is regex-based. Anything past what `render_table` produces (e.g. computed defaults, hybrid properties) won't round-trip.
- There is no auth, no multi-user merge, no diff view. This is intentional.
- `notes`, `links[].hidden`, `colW`, `w/h/wMin/hMin`, and `comment_pinned` are all soft fields — older drafts won't have them and the UI tolerates their absence.

## When something looks wrong

- **Old draft missing buttons / can't delete a table** → check that `normalize(state)` ran on load and back-filled ids.
- **Link line floating in wrong place** → `renderLinks` was probably called without `applying` zoom/pan to `#linkSvg`'s transform. Look at the SVG `style.transform` line near the top of `renderLinks`.
- **Export produces garbage type** → add the type to both `render_col_type` (Python emit) and `_denorm_type` (JSON canonicalize).
- **Frontend stops rendering after a UI change** → almost always a thrown error in the event handler that's silently swallowed. Open the browser console first.
